Instrucciones

Preguntas

  1. ¿Qué es el DOM?

    Respuesta ChatGPT: El DOM (Document Object Model) es una representación estructurada del documento HTML en forma de árbol. Cada parte del documento (etiquetas, texto, atributos, etc.) se convierte en un nodo, y JavaScript puede interactuar con ellos para leer o cambiar contenido, estilos, estructura, etc.

    Respuesta Copilot: El DOM (Document Object Model) es una interfaz de programación que permite a los lenguajes de programación acceder y manipular la estructura, estilo y contenido de un documento HTML o XML. El DOM representa el documento como una jerarquía de nodos, donde cada nodo corresponde a una parte del documento.

    ¿Es correcto? De acuerdo a la documentación de MDN, ambas definiciones son acertadas pero algo ambiguas. Ambas IA's tratan de dar un concepto general del DOM, pero no lo explican a detalles. Esto es probablemente porque no se les pidió un ejemplo o una explicación más detallada. A pesar de esto, la respuesta de ChatGPT parece ser más clara, lo que permite que sea más fácil de comprender, mientras que la de Copilot es más técnica.

    Fuente: ¿Qué es el DOM?

  2. ¿Qué es un nodo?

    Respuesta ChatGPT: Un nodo es una unidad básica dentro del árbol del DOM (Document Object Model), que representa cualquier parte de una página HTML, como elementos, textos, atributos o comentarios. Cada parte del documento se convierte en un nodo para que pueda ser manipulado mediante JavaScript.

    Respuesta Copilot: Un nodo es un objeto que representa un único punto dentro de la estructura del DOM. Puede ser un elemento HTML, un texto dentro de un elemento, un atributo, un comentario, entre otros. Los nodos son los componentes básicos que conforman el árbol del DOM.

    ¿Es correcto? Las respuestas son muy similares y acertadas. De acuerdo a la documentación de MDN, un nodo es un punto único en un árbol del DOM. En este caso, la documentación no ofrece una definición tan clara como la del DOM, lo que hace que las respuestas de las IA's sean más acertadas para personas que no están familiarizadas con el concepto. En este caso, ambas respuestas fueron claras y concisas.

    Fuente: Node (DOM)

  3. ¿Cómo se accede a un elemento del DOM?

    Respuesta ChatGPT: Para acceder a un elemento del DOM se utilizan métodos de JavaScript como getElementById, getElementsByClassName, getElementsByTagName, querySelector o querySelectorAll. Estos métodos permiten seleccionar y trabajar con uno o varios elementos del documento HTML.

    Respuesta Copilot: Para acceder a un elemento del DOM, puedes usar métodos como document.getElementById, document.getElementsByClassName, document.getElementsByTagName, document.querySelector, y document.querySelectorAll. Estos métodos permiten seleccionar elementos específicos o grupos de elementos en el documento HTML.

    ¿Es correcto? Ambas respuestas son correctas y ofrecen una explicación general de cómo acceder a los elementos del DOM, mencionando algunos ejemplos comunes. Comparando las respuestas con la documentación de MDN, esta última ofrece una explicación detallada y con una lista más completa de todos los métodos que se pueden usar para acceder a los elementos del DOM. En este caso, la respuesta de las IA's a forma de resumen es correcta.

    Fuente: ¿Cómo se accede al DOM?

  4. ¿Cómo se modifica un elemento del DOM?

    Respuesta ChatGPT: Para modificar un elemento del DOM, primero se accede a él mediante alguno de los métodos mencionados anteriormente y luego se alteran sus propiedades o contenido, como element.innerHTML para cambiar el contenido interno, element.style para modificar estilos CSS o element.setAttribute para editar atributos.

    Respuesta Copilot: Para modificar un elemento del DOM, primero se selecciona utilizando métodos como document.getElementById o document.querySelector. Luego, se pueden cambiar sus propiedades como innerHTML para el contenido, style para los estilos, o setAttribute para los atributos. También se pueden agregar o eliminar clases con classList.add y classList.remove.

    ¿Es correcto? Amabs respuestas son correctas, pero carecen de ejemplos y una explicación más clara. La respuesta de Copilot es más completa ya que menciona la posibilidad de agregar o quitar clases. En este caso, la documentación sigue ofreciedo una explicación más completa y detallada para modificar elementos.

    Fuente: Element.setAttribute

  5. ¿Cuál es la diferencia entre getElementby* y querySelector*?

    Respuesta ChatGPT: La diferencia entre getElementBy* y querySelector* está en su forma de selección y resultado. Los métodos getElementById, getElementsByClassName y getElementsByTagName permiten seleccionar elementos por ID, clase o etiqueta, y devuelven un solo elemento o una colección HTML. En cambio, querySelector y querySelectorAll utilizan selectores CSS, lo que permite una mayor flexibilidad; querySelector devuelve el primer elemento que coincida y querySelectorAll devuelve todos los que coincidan en forma de NodeList.

    Respuesta Copilot: La diferencia entre getElementBy* y querySelector* radica en cómo seleccionan elementos y el tipo de resultado que devuelven. Los métodos getElementById, getElementsByClassName, y getElementsByTagName seleccionan elementos por ID, clase o etiqueta respectivamente. getElementById devuelve un único elemento, mientras que los otros devuelven colecciones HTML. Por otro lado, querySelector y querySelectorAll utilizan selectores CSS, lo que permite una selección más flexible. querySelector devuelve el primer elemento que coincide con el selector, mientras que querySelectorAll devuelve todos los elementos coincidentes en forma de NodeList.

    ¿Es correcto? Ambas respuestas son correctas, aunque algo técnicas y poco amigables para alguien que no está familiarizado con los conceptos. A pesar de esto, no se menciona cuándo se debe usar cada uno de los métodos, lo que si es explicado de mejor forma en fuentes como Built In.

    Fuente: QuerySelector() frente a GetElementById() en JavaScript

  6. ¿Qué es un evento?

    Respuesta ChatGPT: Un evento es cualquier acción que ocurre en el navegador y que puede ser detectada por JavaScript, como un clic del usuario, una tecla presionada, el envío de un formulario o el cambio de una entrada.

    Respuesta Copilot: Un evento es una acción o suceso que ocurre en el navegador, como un clic, el movimiento del mouse, la carga de una página, o la interacción del usuario con un formulario. Los eventos permiten que JavaScript reaccione a estas acciones y ejecute código en respuesta.

    ¿Es correcto? Ambas respuestas ofrecen una explicación general de lo que es un evento, pero no ofrecen ejemplos de cómo se utilizan y su relación con el DOM. En este caso, las respuestas fallaron en su propósito de explicar el concepto de eventos en el contexto del DOM.

    Fuente: Introducción a los eventos

  7. ¿Qué es un listener?

    Respuesta ChatGPT: Un listener o "event listener" es una función o método que se asigna a un elemento del DOM para que escuche y responda a un evento específico. Se utiliza addEventListener para establecer qué tipo de evento se debe escuchar y qué función se ejecutará cuando ocurra ese evento.

    Respuesta Copilot: Un listener, o "event listener", es una función que se asocia a un elemento del DOM para escuchar eventos específicos, como clics, movimientos del mouse o teclas presionadas. Se utiliza el método addEventListener para registrar un listener, especificando el tipo de evento y la función que se ejecutará cuando ocurra dicho evento.

    ¿Es correcto? Las respuestas son correctas, aunque para este punto carecen de ejemplos y una explicación más clara. En este caso, la respuesta de Copilot es más completa ya que menciona ejemplos de eventos que pueden ser escuchados. A pesar de esto, la documentación de MDN ofrece una explicación más completa y detallada de lo que es un listener y cómo se utiliza.

    Fuente: element.addEventListener

  8. ¿Qué es un callback?

    Respuesta ChatGPT: Un callback es una función que se pasa como argumento a otra función para que sea ejecutada después de que cierta acción haya ocurrido o se haya completado. En el contexto de eventos, los callbacks son las funciones que se ejecutan cuando el listener detecta que ocurrió un evento.

    Respuesta Copilot: Un callback es una función que se pasa como argumento a otra función y se ejecuta después de que la función principal haya completado su tarea. En el contexto de eventos, los callbacks son las funciones que se ejecutan en respuesta a un evento específico, como un clic o una tecla presionada.

    ¿Es correcto? Ambas respuestas son correctas y ofrecen una explicación general de lo que es un callback. Sin embargo, no ofrecen ejemplos de cómo se utilizan y su relación con el DOM. En este caso, las respuestas fallaron en su propósito de explicar el concepto de callbacks en el contexto del DOM. Nuevamente, la documentación de MDN ofrece una explicación más completa y detallada.

  9. ¿Qué es un objeto de evento?

    Respuesta ChatGPT: Un objeto de evento es un objeto que se crea automáticamente cuando ocurre un evento y contiene información relacionada con ese evento, como el tipo de evento, el elemento que lo disparó, coordenadas del mouse, teclas presionadas, entre otros datos útiles para manejar la interacción.

    Respuesta Copilot: Un objeto de evento es un objeto que se genera automáticamente cuando ocurre un evento en el DOM. Este objeto contiene información detallada sobre el evento, como el tipo de evento, el elemento que lo desencadenó, las coordenadas del mouse (si aplica), las teclas presionadas, y otros datos relevantes que permiten manejar el evento de manera precisa.

En este caso, la documentación de MDN ofrece una explicación más completa, aunque no tan clara como la de las IA's.

Conclusiones

Experiencia general

Esta actividad me pareció interesante ya que pude ver las diferencias entre el uso de diferentes IA's en un contexto de investigación y programación. De acuerdo a lo que pude observar, ChatGPT y Copilot son herramientas que si bien no ofrecen explicaciones tan detalladas, te pueden dar una introducción general a los temas para que pueda empezar a investigar.

También tuve un par de complicaciones al utilizar ambas IA's. Por un lado, tuve que especificarle a ChatGPT que necesitaba su respuesta a forma de párrafo para esta actividad, pero en lugar de darme únicamente el texto, me daba todo en formato HTML, lo que no era lo que le estaba especificando. Tuve que hacer múltiples conversaciones para que finalmente me diera una respuesta que me sirviera.

En cuanto a las respuestas, pude ver que eran muy generales, a pesar de pedir que fueran algo más detalladas. Por otro lado, Copilot basaba sus respuestas en las de ChatGPT, por lo que aunque a veces eran algo más específicas, decían en sí lo mismo, ya que únicamente parafraseaba lo demás.

Otro de los aspectos que puedo mencionar que me parecieron interesantes es que, al momento de pedirle a Copilot que generara una base para el archivo .css de la página web, me daba colores que no combinaban o, en general, errores de estilos entre secciones. Al pedirle comentarios para comprender lo que hacía en el código, procedió a agregar un comentario por línea, lo que no es práctico y solo hace el código más difícil de comprender. En este aspecto, considero que Copilot no fue una herramienta útil.

¿Cuándo y para qué usarías ChatGPT y cuándo Copilot?

De acuerdo a lo que pude observar, ChatGPT es mejor al momento de interpretar las instrucciones dadas, siendo más flexible al momento de utilizar esta IA como herramienta. De la misma forma, las respuestas de ChatGPT fueron -en general- más claras y tenían más ejemplos que las de Copilot. En este sentido, pude notar que Copilot ofrece respuestas más técnicas y orientadas a la programación, lo que tiene sentido ya que ha sido diseñada para eso, mientras que ChatGPT fue diseñada para propósito general.

De esta forma, yo usaría ChatGPT cuando necesite comprender algún tema con ejemplos sencillos. La usaría para empezar a investigar sobre los temas y tener una idea general de conceptos nuevos.

Por otro lado, usaría Copilot cuando necesite ayuda comprendiendo errores en mi código para posteriormente solucionarlos por mi cuenta. Personalmente, no me gusta programar con Copilot, ya que muchas veces el autocompletado puede ser algo molesto. Una de las cosas que más me gusta de la programación es ser yo quien resuelva los problemas, por lo que limito el uso de esta IA a entender secciones complejas de códigos o ver la forma en la que la IA implementaría una solución a un problema. Partiendo de esto, ya puedo empezar a ver cómo solucionarlo a mi manera.

Ventajas de usar AI para programar

Personalmente, opino que usar IA's para programar es un arma de doble filo, pero que si puede llegar a ser muy útil si se usa correctamente, como una herramienta y no como el programador. Algunas de las ventajas que he encontrado al momento de usar IA's son las siguientes:

  • Encontrar errores en códigos
  • Comprender bloques de código complejos
  • Automatizar tareas tediosas de programar (como copiar y pegar etiquetas html)
  • Generar ejemplos de código
  • Mejorar la comprensión del código
  • Sugerir correcciones para el código
  • Ayuda a detectar errores de seguridad. Ej: Si usas una API KEY en tu código fuente, te advierte
  • Te puede ayudar a traducir bloques de código a otros idiomas

Desventajas de usar AI para programar

Considero que las IA's aún cuentan con muchas desventajas, sobre todo al momento de programar. Algunas de estas desventajas son las siguientes:

  • Puede hacer que los programadores dependan de la IA para programar
  • Se puede generar código con errores o poco optimizado
  • La IA no siempre entiende el contexto del proyecto en el que la estás usando. Si bien se puede entrenar o darle contexto con archivos, una IA nunca podrá ver el proyecto en ejecución para solucionar los errores.
  • La IA no puede ver la ejecución del código en tiempo real, por lo que no permite solucionar errores de forma creativa. Muchas veces se necesita de pensar fuera de lo que hay en el código para solucionar problemas.
  • La IA puede generar diferentes bloques de código que no son consistentes los unos con los otros
  • Puede llegar a obstaculizar el aprendizaje. Esto ocurre sobre todo con el autocompletado, ya que no permite que pienses por tu cuenta, sino que te ofrece directamente soluciones sin antes verificar que sean adecuadas para el proyecto.
  • Si no se verifica, el código generado por IA puede ser completamente erróneo o no funcionar como se espera.
  • Las IA's se confunden al momento de trabajar en proyectos con muchos archivos, siendo incapaces de ayudarte a identificar errores o solucionar problemas básicos.
