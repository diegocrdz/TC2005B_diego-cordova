Flujo de trabajo de Git/GitHub

Mientras que Git es un sistema de control de versiones, GitHub es un sitio web en el cual se pueden crear repositorios. A través de estos repositorios se puede colaborar y trabajar en equipo, tener diferentes versiones de un mismo proyecto, entre otros.

El flujo de trabajo de Git/GitHub puede ser representado de la siguiente forma:

  1. Clonar un repositorio de GitHub.
  2. Realizar cambios en los archivos.
  3. Subir los cambios al repositorio local con Git.
  4. Subir los cambios al repositorio remoto con Git.

De esta manera, se puede trabajar de forma colaborativa con diferentes versiones de un mismo proyecto.

Git

Git es un sistema de control de versiones. A través de esto, un clon local es un repositorio de control de versiones del proyecto subido a la nube de forma remota. Se puede utilizar para coolaborar o tener diferentes versiones de un mismo proyecto.

Comandos básicos de Git

Comando Descripción
git clone <url> Crea un clon local del repositorio remoto.
git add <archivo> Añade archivos al área de preparación.
git commit -m <comentario> Comenta los cambios realizados. Se suben los cambios al repositorio local.
git push -u origin <rama> Sube todos los cambios ya comentados a la rama main del repositorio en Github.
git pull Descarga los cambios del repositorio remoto al local.
git status Muestra el estado de los archivos en el repositorio.
git log Muestra el historial de commits.
git checkout -b <rama> Crea una nueva rama y se cambia a ella.
git branch -a Muestra las ramas existentes.
git branch -d <rama> Elimina la rama indicada.
git branch -D <rama> Forzar el borrado de la rama indicada.
git merge <rama> Combina los cambios de una rama con la rama actual.
git switch <rama> Cambia a la rama indicada.
git restore <archivo> Descarta los cambios realizados en un archivo.
git fetch -a Descarga los cambios del repositorio remoto al local.

GitHub

GitHub es un sitio web mediante el cual se pueden crear repositorios; es decir, un lugar en el que puedes almacenar archivos y su respectivo historial de revisión.

A través de los repositorios se puede trabajar en equipo, colaborar, trabajar en distintas revisiones o versiones de un mismo archivo, etc. Finalmente, se puede subir, descargar o clonar el repositorio (mediante comandos en Git o integraciones de Visual Studio Code).

Para clonar un repositorio de GitHub, se debe copiar la URL del repositorio y ejecutar el comando git clone <url> en la terminal.
